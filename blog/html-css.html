Difference between Relative, Absolute and Fixed Positioning

Last updated: 25 November 2021

Absolute

This is a type of positioning that allows you to place any page element precisely where you want it. You use the positioning attributes top, left, bottom, and right to set the location. Remember that these values will be relative to the next parent element with relative (or absolute) positioning. If there is no such parent, it will default all the way back up to the element itself meaning it will be placed relative to the page itself. The most important thing about absolute positioning is that these elements are removed from the flow of elements on the page. An element with this type of positioning is not affected by other elements and it doesnâ€™t affect other elements. This means that use of this sort of positioning can limit the flexability and interoperability of your page.

Relative

A relative element is positioned relative to its normal position. This element will be adjusted away from its default position on the page if its top, right, bottom, and left properties are set. Any gaps left by the element will not be filled with other content.

Fixed

A fixed position element is one that is positioned relative to the viewport, or the browser window. The viewport does not change when the window is scrolled, so a fixed positioned element will remain where it is as the page scrolls. You may use this for something like a navigation bar that needs to remain visible regardless of where the page is scrolled. The challenge with fixed positioning is that it may overlap other content, making it inaccessible - something to consider when making the most of this element type.
