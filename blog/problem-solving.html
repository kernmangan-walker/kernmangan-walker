Some of my favorite Javascript problem solving tools include:


Pseudo code is a methodology that allows the programmer to represent the implementation of their code through simple line-by-line descriptions of what the code is supposed to do. The idea with pseudo code is that they can be interpreted by programmers no matter what their programming background or knowledge is, or familiarity with the project.

A basic example of pseudo code is:
Creating a basic random number generator
function getRandomArbitrary(min, max) {
return Math.random() * (max - min) + min; }

In Pseudocode this would be:
// Define function with array of numbers that the range can be set between
// Use Javascript function math.random to generate the the specific number
// Output generated number

The Rubber Ducky Method is all about explaining code you’ve worked on - describing how it works, what every part of it is supposed to do, allowing you to think about your code more simply and perceive it from a different perspective. The idea is that it would be as if you are speaking to a rubber duck. Explaining something out loud gives you a chance to understand more clearly if it makes perfect sense or not, see potential gaps in your approach and identify how they might be resolved.
For many people this allows them to find solutions to problems and understand what may be contributing to bugs before they go ahead and speak with colleagues - minimising interruption amongst the team while also facilitating problem-solving.

Understanding Error Messages is super useful approach for problem solving with code is to look for and decipher the different error messages you might receive through testing or when your dev environment tries to “run” the code. It’s really important to understand the different types of error messages you might receive as well as knowing where you might be able to look for these errors.

For example, you may receive an error referring to a specific line of code but this isn’t necessarily the origin of the error as the elements on that line might be more specifically tied to other elements elsewhere in your document. Usually by looking into the origin of the variables/function/other elements on the line with the related error, you can have a look into where the error might actually originate.
The three main types of errors that can occur with coding Javascript are:

Logical Errors: Apparently one of the most common error types, it relates to a bug with the program that causes it to crash. Effectively the program is operating not as intended at all. It can be very difficult to trace as the program can completely crash or exit, without clear directions to the error in question.

Syntax Errors: Relate to an issue with the syntax, characters or formatting you are using. Basically means that the program is unable to read the code you’ve written and won’t properly execute.

Runtime Errors: This relates to code that the program (or interpreter) cannot understand. In these instances the syntax might exist but there may be an element of code that is missing or in not included correctly within an object that was intended to be executed.
